# App Store Screenshots

`fastlane deliver` expects screenshots organized per locale, with Apple's
device size naming. Target layout:

```
fastlane/screenshots/ios/
├── en-US/
│   ├── 1_APP_IPHONE_67_home.png           # 6.7" — 1290×2796 (iPhone 15 Pro Max)
│   ├── 2_APP_IPHONE_67_river_detail.png
│   ├── 1_APP_IPHONE_65_home.png           # 6.5" — 1284×2778 (legacy fallback)
│   └── ...
└── fr-CA/
    └── ...
```

Apple currently requires one set of screenshots for the largest iPhone size
you support (6.7"/6.9"), and a second set for the 6.5" size if you target
older devices. iPad screenshots are only needed if the app ships on iPad.

Screenshots are not regenerated by a committed script yet. When wiring one
up, mirror `scripts/generate-store-screenshots.ts` but swap the Playwright
device descriptor to an iPhone profile (`devices["iPhone 15 Pro Max"]`) and
write files under `fastlane/screenshots/ios/<locale>/`.
